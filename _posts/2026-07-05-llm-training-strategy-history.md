> 一句话总结：大模型训练策略的发展，就是从“让模型会续写”，一路走到“让模型听得懂人话、符合人类偏好，并能靠可验证奖励提升推理能力”。

![大模型训练策略时间线](/assets/images/training-strategy/training-strategy-timeline.svg)

<!--more-->

如果你把大模型想象成一个学生，那它的成长路径其实挺像人类学习：先大量读书，学会语言和知识；然后看标准答案，学会怎么回答问题；再接受老师和用户的反馈，知道什么回答更好；最后开始刷数学题、跑代码测试，通过明确的对错信号提升推理能力。

这条路线背后，对应的就是大模型训练策略的演进：预训练、SFT、RLHF、PPO、DPO、GRPO，以及最近很火的推理强化学习。

## 1. 预训练：先让模型学会“接话”

最早的大模型训练，核心目标非常朴素：预测下一个 token。

```text
输入：今天天气很
目标：好
```

模型在海量文本上反复做这件事，慢慢学会语言规律、常识、事实知识、代码模式，甚至一些推理能力。GPT、LLaMA、Qwen、DeepSeek、Mistral 这些基础模型，最开始都要经历这个阶段。

预训练的好处是规模化很强，只要有足够多的数据和算力，模型就能越学越强。但问题也很明显：它学会的是“续写”，不是“当一个好助手”。

所以早期基础模型经常会出现一种尴尬状态：它很会说话，也知道很多东西，但不一定按你的要求回答。你让它总结文章，它可能继续续写文章；你让它解决问题，它可能开始模仿网页格式。

换句话说，它像一个读了很多书的人，但还没学会怎么跟用户合作。

## 2. SFT：让模型学会听指令

下一步就是 SFT，Supervised Fine-Tuning，监督微调。

SFT 的做法是准备大量“指令-回答”样本，让模型模仿高质量答案：

```text
用户：帮我写一封请假邮件。
助手：好的，下面是一封可以参考的请假邮件……
```

这个阶段非常关键。它让模型从“文本续写器”变成“指令助手”。模型开始知道用户问问题时，自己应该回答；用户让它写代码，它应该给代码；用户让它总结，它应该提炼重点。

ChatGPT 和 InstructGPT 的成功，很大程度上就来自这种指令微调思路。OpenAI 的 InstructGPT 论文里就展示过：经过指令微调和人类反馈训练后，较小模型在人类偏好上也可以超过更大的原始 GPT-3。

但 SFT 也有局限。它本质上是在模仿标准答案，而现实里很多问题并没有唯一标准答案。比如：

```text
哪种回答更有帮助？
哪种回答更安全？
哪种回答更符合用户真实意图？
哪种解释更清楚？
```

这些问题光靠“模仿答案”不够，还需要偏好反馈。

## 3. RLHF + PPO：让模型学会“人类更喜欢什么”

于是 RLHF 出场了。RLHF 全称是 Reinforcement Learning from Human Feedback，也就是人类反馈强化学习。

经典 RLHF 流程一般分三步：

```text
1. SFT：先用人工示范训练一个初始助手模型
2. Reward Model：让人类比较多个回答，训练奖励模型
3. PPO：用强化学习优化模型，让它获得更高奖励
```

比如同一个问题，模型生成三个回答 A、B、C。标注员看完之后说：B 最好，A 一般，C 最差。我们就可以用这些偏好数据训练一个 Reward Model，让它学会预测“人类更喜欢哪个回答”。

接下来，模型继续生成回答，Reward Model 给分，训练算法再推动模型生成更高分的答案。

这里常用的强化学习算法就是 PPO。

![RLHF 与 PPO 流程](/assets/images/training-strategy/rlhf-ppo-flow.svg)

PPO，全称 Proximal Policy Optimization，最早是一个通用强化学习算法。它的核心思想可以轻松理解为：模型可以变好，但别一下子变得太离谱。

也就是说，训练目标大概是：

```text
一方面：让奖励模型给更高分
另一方面：不要偏离原来的 SFT 模型太远
```

这个“不要偏太远”很重要。否则模型可能为了讨好奖励模型，学出奇怪套路，比如废话变多、过度迎合、格式浮夸，甚至出现 reward hacking。

PPO-RLHF 的优点是效果强，也是早期对齐 ChatGPT 类模型的经典路线。缺点也很硬：训练复杂、成本高、调参麻烦，还要额外训练 Reward Model 和 value model，工程上并不轻松。

## 4. RLAIF：让 AI 帮人类打工

RLHF 很好，但人类标注太贵。于是大家自然会想：能不能让 AI 来做一部分反馈？

这就是 RLAIF，Reinforcement Learning from AI Feedback。

Anthropic 的 Constitutional AI 是一个代表性方向。它会给模型一组原则，比如回答要有帮助、诚实、无害。然后让 AI 根据这些原则批改回答、比较回答、生成偏好数据。

流程大概是：

```text
人类写原则
AI 按原则评价回答
模型根据 AI 反馈继续优化
```

这条路线的好处是能降低人工标注成本，也更容易批量扩展。问题是，AI 的判断不一定永远可靠。如果 AI 评价本身有偏差，模型也可能沿着偏差继续学。

所以 RLAIF 不是完全替代人类，而更像是把人类从“每道题都批改”升级为“制定原则、抽查质量、修正规则”。

## 5. DPO：不用 PPO，也能做偏好优化

到了 2023 年，DPO 开始流行。

DPO，全称 Direct Preference Optimization，直接偏好优化。它的想法非常优雅：既然我们已经有人类偏好数据，比如：

![DPO 偏好优化示意图](/assets/images/training-strategy/dpo-preference.svg)

```text
chosen：更好的回答
rejected：较差的回答
```

那为什么还要训练一个 Reward Model，再用复杂的 PPO 去优化呢？能不能直接让模型提高 chosen 的概率，降低 rejected 的概率？

DPO 的答案是：可以。

它把 RLHF 的目标改写成一个更像监督学习的损失函数。训练时不需要单独的 Reward Model，也不需要在线采样和 PPO 循环。工程体验一下子好了很多。

这也是为什么很多开源模型喜欢用 DPO、IPO、KTO、ORPO、SimPO 这类方法。它们大多可以看作“更简单的偏好优化路线”：不用完整 RLHF，也能让模型更符合人类偏好。

DPO 的优点很明显：

1. 训练更简单。
2. 稳定性更好。
3. 成本更低。
4. 很适合开源模型后训练。

但它也不是万能的。DPO 更像是在已有偏好数据上做离线学习，如果你希望模型在训练中自己探索新解法，尤其是数学、代码、复杂推理任务，DPO 的探索能力就不够了。

这就引出了下一阶段：推理模型的强化学习。

## 6. 推理模型时代：奖励不只是“人类喜欢”，而是“答案真的对”

传统 RLHF 主要优化的是：

```text
这个回答是否有帮助？
这个回答是否安全？
这个回答是否符合人类偏好？
```

但数学、代码、逻辑推理这类任务有一个特别好的地方：它们往往可以自动验证。

数学题可以检查最终答案，代码题可以跑单元测试，格式题可以用规则判断。也就是说，我们不一定要让人类来判断“喜不喜欢”，而是可以直接判断“对不对”。

这件事非常重要。

因为它让强化学习从“讨好人类偏好”变成了“追求可验证的正确性”。模型不只是学会回答得更舒服，而是学会真的把题做对。

DeepSeek-R1-Zero 就是这个方向的代表之一。它展示了一个很有意思的现象：如果直接对基础模型做大规模强化学习，并使用准确性、格式等可验证奖励，模型会自然出现更长的推理链、自我反思、检查答案、重新尝试等行为。

这意味着 RL 不只是对齐工具，也可以成为能力增强工具。

## 7. GRPO：更轻量的推理强化学习

说到 DeepSeek-R1，就绕不开 GRPO。

GRPO，全称 Group Relative Policy Optimization，组相对策略优化。它可以理解为 PPO 的一个更适合大模型推理训练的轻量版本。

传统 PPO 通常需要一个 value model 或 critic，来估计某个回答“比预期好多少”。但在大模型训练里，再维护一个 value model 很贵，也容易不稳定。

GRPO 的思路更直接：同一个问题，一次采样多个回答，然后在这一组回答内部做相对比较。

![GRPO 组内相对奖励示意图](/assets/images/training-strategy/grpo-group-relative.svg)

比如同一道数学题，模型生成 8 个解答：

```text
回答 1：错，奖励 0
回答 2：对，奖励 1
回答 3：错，奖励 0
回答 4：对，奖励 1
回答 5：错，奖励 0
回答 6：错，奖励 0
回答 7：对，奖励 1
回答 8：错，奖励 0
```

GRPO 不再单独训练 value model，而是用这一组回答的平均表现作为参照。某个回答比组内平均好，就提高它的概率；比组内平均差，就降低它的概率。

粗略理解就是：

```text
优势 = 当前回答奖励 - 同组平均奖励
```

当然真实公式更复杂，还会加入 KL 约束，避免模型偏离参考模型太远。但直觉上，GRPO 就是在让模型进行“同题多解、自我比较、奖励更优路径”。

它的好处是：

1. 不需要额外的 value model。
2. 更省显存和算力。
3. 很适合数学、代码这类可验证任务。
4. 能鼓励模型探索不同推理路径。

这也是为什么 GRPO 会在推理模型训练里受到关注。它把强化学习变得更适合大规模语言模型，也更适合开源社区复现和改造。

## 8. 现在的主流做法：不是单选，而是组合拳

今天的大模型训练，已经很少只靠一种方法打天下。更常见的是多阶段组合：

```text
预训练
-> 指令 SFT
-> 偏好优化：DPO / IPO / ORPO / KTO
-> 推理 RL：PPO / GRPO
-> 拒绝采样与蒸馏
-> 安全对齐和领域微调
```

如果是面向通用助手，SFT 和偏好优化仍然很重要，因为它们决定模型是否好用、礼貌、清楚、符合人类预期。

如果是面向数学、代码、科学推理，GRPO 这类基于可验证奖励的强化学习会越来越重要，因为它能直接推动模型提高任务成功率。

所以未来训练策略大概会分成两条线：

```text
助手对齐线：让模型更有帮助、更安全、更符合人类偏好
能力增强线：让模型在数学、代码、工具、科学任务中真的做对
```

前者离不开 SFT、RLHF、DPO、RLAIF。后者会越来越依赖可验证奖励、GRPO、过程奖励模型、自动评测环境和推理数据合成。

## 简单对比

| 方法 | 核心思想 | 优点 | 局限 |
|---|---|---|---|
| 预训练 | 预测下一个 token | 学知识和语言能力 | 不一定会听指令 |
| SFT | 模仿高质量示范 | 简单稳定，让模型像助手 | 探索能力弱 |
| RLHF + PPO | 用人类偏好训练奖励模型，再用 RL 优化 | 效果强，经典成熟 | 复杂、贵、不稳定 |
| RLAIF | 用 AI 反馈替代部分人类反馈 | 降低标注成本 | 依赖 AI 评价质量 |
| DPO | 直接用 chosen / rejected 偏好对优化 | 简单、稳定、工程友好 | 更依赖离线数据 |
| GRPO | 同题多采样，用组内相对奖励做 RL | 省掉 value model，适合推理训练 | 依赖可验证奖励和采样质量 |

## 结语

大模型训练策略的发展，其实是在不断回答一个问题：我们到底希望模型变成什么？

预训练回答的是：希望它懂语言、懂知识。  
SFT 回答的是：希望它听得懂指令。  
RLHF 和 PPO 回答的是：希望它更符合人类偏好。  
DPO 回答的是：希望偏好优化更简单、更稳定。  
GRPO 回答的是：希望模型能通过可验证奖励，自己探索出更强的推理能力。

如果说 PPO 代表了 ChatGPT 早期对齐时代，DPO 代表了开源模型后训练的工程化时代，那么 GRPO 则代表了推理模型的新阶段：模型不只是回答得更像人，而是开始在数学、代码和科学任务里，通过明确反馈把自己训练得更会思考。
