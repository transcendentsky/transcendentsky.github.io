# 多模态模型 VLM 的后训练与微调：从视觉指令微调到偏好对齐

多模态模型的能力不是“接一个视觉编码器”就自然出现的。真正决定 VLM 能不能稳定落地的，往往是预训练之后的后训练工程：视觉-语言对齐、视觉指令微调、领域数据微调、偏好优化、grounding 数据、OCR/图表/视频任务强化，以及一整套能发现幻觉和定位错误的评估体系。

一句话总结：

> VLM 后训练的核心，是把“能看见图像特征的语言模型”，训练成“能按人类意图使用视觉证据完成任务的多模态助手”。

<!--more-->

---

## 一、为什么 VLM 的后训练比纯文本 LLM 更复杂

纯文本 LLM 的后训练主要围绕指令遵循、领域知识、输出格式、偏好对齐和安全边界展开。VLM 也需要这些，但还多了几类问题：

- **视觉-语言对齐**：视觉 token 到语言 token 的接口是否稳定，模型是否真的使用图像信息。
- **空间和细粒度感知**：能不能理解位置、数量、关系、局部区域、遮挡和小目标。
- **OCR 与文档理解**：能不能读图中文字、表格、票据、截图和复杂版面。
- **grounding 与引用**：回答是否能落到图像区域、bbox、mask 或证据片段上。
- **多图、视频和时序**：能不能处理跨帧变化、步骤顺序和多图对比。
- **多模态幻觉**：模型是否会无中生有，看不清却说得很确定。

所以 VLM 微调不能只复用 LLM 的 SFT 经验。它必须回答一个更难的问题：模型是在根据图像作答，还是在根据语言先验猜答案？

---

## 二、VLM 后训练的典型阶段

一个常见的 VLM 训练链路可以拆成四段。

```text
视觉编码器 + LLM + 连接层
-> 视觉-语言对齐预训练
-> 视觉指令微调 SFT
-> 偏好优化 / RLHF / DPO
-> 领域微调与部署评估
```

不同模型实现会有差异，但后训练目标大体一致。

### 1. 视觉-语言对齐

这一阶段解决“图像特征怎么进入语言模型”的问题。常见结构包括：

- CLIP/ViT/SigLIP 等视觉编码器。
- projector、MLP connector、Q-Former 或 cross-attention 模块。
- LLM backbone，例如 LLaMA、Qwen、Vicuna、InternLM 等。

早期 LLaVA 的做法很典型：先冻结视觉编码器和语言模型，只训练投影层，让图像 patch 特征映射到 LLM 可理解的 token 空间；随后再做视觉指令微调。LLaVA 1.5 进一步说明，一个简单但设计正确的 MLP connector，加上高质量公开数据，就能形成很强的视觉指令微调基线。

这一阶段的关键不是让模型“会聊天”，而是让语言模型能稳定接收视觉证据。

### 2. 视觉指令微调

视觉指令微调是 VLM 后训练的核心。它把图像、用户指令和理想回答组织成对话数据，让模型学习：

- 看图回答问题。
- 描述图像。
- 解释图表。
- 读文字和截图。
- 做视觉推理。
- 按格式输出结构化结果。
- 在不确定时承认看不清或要求补充信息。

InstructBLIP 的思路是把大量视觉任务转成 instruction format，并使用 instruction-aware Query Transformer，让视觉特征提取受指令影响。这一点很重要：同一张图，用户问“这是什么动物”和“左上角标牌写了什么”，需要关注的视觉区域完全不同。

### 3. 偏好优化与安全对齐

SFT 教模型模仿标准答案，但多模态回答经常没有唯一答案。比如图像描述可以详细也可以简洁，医学影像可以保守也可以直接，截图分析可以给结论也可以列证据。

这时需要偏好数据：

```json
{
  "image": "...",
  "prompt": "这张图里有什么风险？",
  "chosen": "图中地面湿滑，行人可能摔倒。建议设置警示牌。",
  "rejected": "这是一个普通走廊，没有明显问题。"
}
```

偏好优化可以用 DPO、RLHF、RLAIF 或基于规则/模型打分的 reward。VLM 场景下，偏好优化尤其适合处理：

- 幻觉抑制。
- 不确定性表达。
- 安全拒答。
- 医疗、工业、金融等高风险视觉判断。
- 视觉证据优先，而不是语言常识优先。

### 4. 领域微调

通用 VLM 并不等于垂直 VLM。工业巡检、遥感、医疗影像、自动驾驶、机器人、金融票据、网页截图，都有不同的数据分布和错误成本。

领域微调通常要补三类数据：

- **领域视觉数据**：设备缺陷图、病灶图、遥感图、表格截图、UI 截图。
- **领域语言规范**：诊断术语、报告模板、处置建议、合规表达。
- **领域评估集**：难例、边界例、低清晰度、遮挡、反光、异常版式。

如果只是让模型学领域术语，LoRA/QLoRA 往往够用；如果要显著改变视觉感知或 grounding 能力，可能需要调整 connector、视觉 encoder 适配层，甚至继续做多模态预训练。

## 三、VLM 微调数据怎么设计

VLM 数据的难点不在“多”，而在“视觉证据是否真的对答案必要”。

### 1. 数据格式

常见样本格式：

```json
{
  "messages": [
    {"role": "user", "content": [
      {"type": "image", "image": "images/example.jpg"},
      {"type": "text", "text": "请判断图中设备是否存在异常，并给出依据。"}
    ]},
    {"role": "assistant", "content": "图中右侧绝缘子表面有明显污秽..."}
  ]
}
```

好的数据应当包含：

- 明确任务指令。
- 图像证据可验证。
- 回答中有视觉依据。
- 不确定场景有保守表达。
- 格式和部署时一致。

### 2. 任务覆盖

VLM SFT 至少要覆盖以下任务族：

| 任务类型 | 训练目标 | 常见风险 |
| --- | --- | --- |
| Caption | 建立基础图像描述能力 | 容易学成空泛描述 |
| VQA | 回答视觉问题 | 依赖语言先验猜答案 |
| OCR | 读图中文字 | 小字、旋转、低清晰度失败 |
| Chart/Table | 理解图表和表格 | 数值读取和比较错误 |
| Grounding | 输出区域、坐标、证据 | bbox 不准，区域引用漂移 |
| Multi-image | 多图比较、变化检测 | 图像顺序和引用混乱 |
| Video | 时序理解、动作识别 | 抽帧丢失关键事件 |
| Domain QA | 领域视觉诊断 | 过度自信、专业风险 |

### 3. 负样本和拒答样本

很多 VLM 微调只准备“能回答”的样本，却忽略“不能回答”的样本。实际部署中，这会导致模型强行判断。

需要准备：

- 图像缺失但用户要求看图。
- 图片模糊、遮挡、分辨率不足。
- 问题要求识别无法确定的人物身份。
- 用户要求医学/法律/安全高风险结论。
- 图中没有用户提到的对象。
- OCR 文字无法可靠读取。

这类样本的目标不是让模型更会拒绝，而是让模型知道什么时候视觉证据不足。

## 四、VLM 微调时到底训哪些参数

常见选择有四种。

### 1. 只训 projector / connector

适合冷启动对齐，成本低，风险小。缺点是只能解决基础视觉到语言空间映射，不足以学复杂指令和领域行为。

适合：

- 新接视觉 encoder。
- 快速验证架构。
- 小规模 caption / image-text 对齐。

### 2. 训 connector + LLM LoRA

这是很多企业落地最常见的方案。保留视觉 encoder，给 LLM 加 LoRA，必要时 connector 全量训练。

适合：

- 领域 VQA。
- 格式化输出。
- 文档/票据/截图理解。
- 低成本多任务适配。

风险是模型可能学会语言模板，但视觉 grounding 没有真正提升。

### 3. 训视觉 encoder 的高层或 adapter

当任务强依赖视觉细节，比如遥感、医学影像、工业缺陷、小目标检测，只训 LLM 可能不够。此时可以训练视觉 encoder 高层、adapter 或视觉侧 LoRA。

适合：

- 视觉分布和通用图像差异很大。
- 细粒度缺陷识别。
- 专业影像。
- 小目标或局部纹理。

风险是破坏通用视觉能力，训练和评估成本更高。

### 4. 全参数微调

效果上限高，但成本和风险也最高。除非数据量足够大、目标很明确、评估体系成熟，否则不建议直接全参数微调。

## 五、后训练中的几个关键矛盾

### 1. OCR 能力和视觉推理能力不是一回事

很多模型能描述图片，却读不准小字；也有模型 OCR 很强，但复杂图表推理弱。训练时要拆开评估：

- 文本识别是否正确。
- 数值读取是否正确。
- 表格结构是否正确。
- 基于数值的推理是否正确。

不要用一个总分掩盖问题。

### 2. 视觉幻觉来自两个方向

一种是语言先验太强：用户问“这个人戴眼镜吗”，模型没看清也猜。另一种是训练数据太模板化：模型学会了“工业图像就说有裂纹”“医学图像就说建议就医”。

缓解方式：

- 加入反事实样本。
- 加入无异常样本。
- 加入看不清样本。
- 要求回答引用视觉证据。
- 使用 grounding 或区域标注监督。

### 3. 高分 benchmark 不等于可部署

通用榜单通常覆盖 VQA、OCR、数学、图表、幻觉等能力，但业务落地还要看：

- 输入图像来自真实相机还是清洗数据集。
- 是否存在压缩、模糊、遮挡、反光。
- 是否需要严格格式。
- 是否要低延迟。
- 是否允许错误建议。
- 是否需要可解释证据。

VLM 部署前必须做业务闭环评估。

### 4. 图像分辨率和 token 成本要一起考虑

Qwen2-VL 这类模型引入动态分辨率和多模态位置编码，说明一个现实问题：VLM 不是分辨率越高越好。高分辨率能保留细节，但会带来更多视觉 token、显存、延迟和上下文压力。

工程上要权衡：

- OCR 和小目标需要高分辨率。
- 粗粒度场景理解可以低分辨率。
- 长文档和视频要做裁剪、分页、抽帧或检索。
- 对 UI/票据/图表，局部放大往往比整图硬塞更有效。

## 六、VLM 后训练的推荐流程

一个务实流程如下：

```text
1. 定义目标场景和失败成本
2. 选基础 VLM 和输入格式
3. 建立分层评估集
4. 收集真实图像和问题
5. 构造视觉证据充分的 SFT 数据
6. 训练 connector + LoRA
7. 做 badcase 归因
8. 补 OCR / grounding / 拒答 / 难例数据
9. 加偏好优化抑制幻觉和过度自信
10. 小流量上线和持续回收样本
```

这里最重要的是第 3 步。没有评估集，微调很容易变成“回答看起来更像了”，但并不知道它是否更可靠。

建议评估集分成：

- 通用多模态能力集。
- OCR/文档专项集。
- 图表/表格专项集。
- 视觉幻觉集。
- 业务真实样本集。
- 高风险拒答集。
- 格式化输出校验集。

## 七、什么时候该 SFT，什么时候该偏好优化

可以用一句话区分：

> SFT 解决“应该怎么回答”，偏好优化解决“多个回答里哪个更好”。

适合 SFT：

- 输出格式不稳定。
- 领域术语不会用。
- OCR 结果要转结构化字段。
- 图像诊断流程要固定。
- 多轮视觉问答要遵循模板。

适合偏好优化：

- 回答正确但不够谨慎。
- 模型容易幻觉。
- 答案过长或过短。
- 需要按视觉证据排序。
- 高风险场景需要更保守。
- 用户偏好难以写成唯一标准答案。

偏好优化不是 SFT 的替代品。通常先用 SFT 建立能力和格式，再用 DPO/RLHF/RLAIF 调整偏好、安全和不确定性表达。

## 八、常见排障

### 1. 微调后模型不看图了

可能原因：

- 文本问题和答案本身就能猜出结果。
- 训练样本视觉证据不必要。
- 图像 token 被截断或模板错误。
- loss mask 错误。
- LoRA 学到语言捷径。

解决：加入需要图像才能回答的样本、做图像打乱测试、加入反事实图像对。

### 2. OCR 一直不好

可能原因：

- 分辨率太低。
- 视觉 encoder 不适合文字。
- 数据缺少真实截图/文档。
- 标注没有保留版面结构。

解决：提高局部分辨率、做 crop、引入 OCR 专项数据、结合外部 OCR 工具。

### 3. 模型定位不准

可能原因：

- 没有 bbox/mask/区域引用数据。
- 训练只要求自然语言回答。
- 坐标格式不统一。

解决：统一坐标格式，加入 grounding 数据，回答中要求引用区域证据。

### 4. 领域微调后通用能力下降

可能原因：

- 数据太窄。
- epoch 太多。
- 学习率过大。
- 没有混入通用视觉指令数据。

解决：降低学习率、early stopping、混入通用样本、使用 LoRA 而不是全参数。

## 九、总结

VLM 后训练的关键不是简单“拿图文数据再训一下”，而是让模型学会可靠地使用视觉证据。

最值得记住的几点：

- 视觉-语言对齐解决“图像怎么进入 LLM”。
- 视觉指令微调解决“模型怎么按任务使用图像”。
- 偏好优化解决“回答是否谨慎、可靠、符合人类偏好”。
- 领域微调要区分语言适配和视觉感知适配。
- VLM 评估必须拆开 OCR、grounding、推理、幻觉、格式和业务真实样本。
- 如果训练数据不要求模型真正看图，模型就会学会用语言先验猜答案。

VLM 的微调上限，最终取决于三件事：视觉证据质量、任务监督粒度，以及是否能持续从真实 badcase 中修正模型。
